# AGENTS.md

## Project Conventions

- This project is a native Swift macOS app named `StarMagpie`.
- Xcode project configuration is managed by XcodeGen. Run `xcodegen generate` after changing `project.yml`.
- Main stack: SwiftUI, SwiftData, URLSession, and Security Keychain.
- App strings use English source text as the base language. Localization is provided through `en.lproj` and `zh-Hans.lproj`, and the app supports Follow System, English, and Simplified Chinese language options.
- `README.md` is the canonical English README. `README.zh-CN.md` is the Simplified Chinese translation.
- This is a personal open source project licensed under GNU GPL v3. The license text is in `LICENSE`.
- When user-facing behavior, installation instructions, contribution flow, or license notes change, update both `README.md` and `README.zh-CN.md`.
- Public open source collaboration docs, issue templates, PR templates, CI, and repository governance files should be written in English and live under `.github/` when applicable.

## Common Commands

```bash
xcodegen generate
xcodebuild test -scheme StarMagpie -destination 'platform=macOS'
xcodebuild build -scheme StarMagpie -destination 'platform=macOS'
./scripts/generate-app-icon.swift
./scripts/package-unsigned.sh
```

## Development Notes

- Do not write GitHub tokens to SwiftData or plain files. Use Keychain only.
- GitHub Stars API requests must keep `Accept: application/vnd.github.star+json` to receive `starred_at`.
- Sync merge uses GitHub repo `id` as the primary key and preserves local fields: `manualCategoryId`, `notes`, and `lastViewedAt`.
- Repository import/export uses StarMagpie JSON archives. Imports merge by GitHub repo `id` and do not delete local repositories missing from the archive.
- When adding user-facing text, update both `en.lproj/Localizable.strings` and `zh-Hans.lproj/Localizable.strings`.
- Dynamic strings, error messages, category names, and sort names must use `AppLocalizer` so in-app manual language switching takes effect immediately.
- Security issues must follow `SECURITY.md`. Do not expose tokens, secrets, or private repository data in public issues, logs, screenshots, or test fixtures.
- The project does not currently use an Apple Developer certificate. Release packages are generated by `scripts/package-unsigned.sh` as zip files without Developer ID signing or notarization. Do not commit signing certificates, provisioning profiles, or notarization credentials.
